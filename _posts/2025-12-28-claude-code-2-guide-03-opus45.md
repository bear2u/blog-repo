## Opus 4.5가 좋은 느낌인 이유

스크린샷에서 설명했듯이, Opus 4.5는 GPT-5.1-Codex-Max와 대략 같은 코드 생성 능력을 가졌습니다.

오늘날, 제 경험상 GPT-5.2-Codex가 Opus 4.5를 작은 마진으로 원시 능력에서 초과한다고 생각합니다. 그럼에도 불구하고, Opus 4.5는 출시 이후 제 메인 드라이버였습니다.

첫 번째 이유는 **더 빠르고** Codex보다 훨씬 적은 시간에 비슷한 난이도의 작업을 수행할 수 있다는 것입니다. 또한, Codex보다 전반적으로 훨씬 더 나은 **커뮤니케이터이자 페어 프로그래머**입니다. Codex는 때때로 지시를 무시하고 변경을 할 수 있습니다. Opus는 **의도 감지**도 더 좋습니다.

Thariq가 보여준 좋은 사용 사례 중 하나는 Claude의 설명 능력을 활용하여 비기술적인 사람에게 변경 사항을 설명하는 백그라운드 비동기 에이전트를 만드는 것입니다.

## Claude vs Codex 비교

차이를 더 보여주기 위해, CC vs Codex 비교입니다:

### Claude 출력 특성
- 대화체 톤
- 상세한 설명
- 높은 대비 텍스트와 더 굵은 폰트
- 기대에 부합하는 출력

### Codex 출력 특성
- 항상 중첩된 글머리 기호로 작성
- 더 간결함
- 더 얇은 텍스트, 읽기 어려움
- 생각 흔적이 더 연한 색상으로 표시되어 눈이 피로함

같은 프롬프트에 대해 출력을 비교하면, Codex는 여전히 조금 더 간결한 반면 Claude는 제 기대에 맞습니다. (Codex에서 "reveal your thoughts in detail" 같은 것을 추가하면 더 자세히 쓰게 할 수 있습니다.)

## 빠른 피드백 루프의 마법

**작업 수행에 필요한 사고 시간뿐만 아니라 처리량 면에서도 더 빠르기 때문에, 작업에 대한 훨씬 빠른 피드백 루프가 가능해집니다.** 이것은 능력 면에서 GPT-5.1/Codex가 11월에도 동등했음에도 불구하고 진행이 더 실감나게 느껴지게 합니다. 빠른 루프의 유일한 단점은 조심스럽다면 오랜 시간 마이크로 매니징을 하게 된다는 것입니다.

## Opus 4.5의 글쓰기 능력과 개성

Opus 4.5는 훌륭한 작가이고 인간에 가장 가깝기 때문에 프롬프트 커스터마이징에 항상 Claude 모델을 선호해왔습니다.

제가 주장하는 것은 아니지만, 많은 사람들이 Claude Opus 4.5의 개성과 말하는 방식을 좋아합니다 - 일부는 Opus 4.5가 **"영혼(soul)"**이 있다고 언급합니다. 이 특성은 Sonnet 3.7, Sonnet 4, Opus 4, Opus 4.1에서 다소 적었지만 **Opus 4.5에서 돌아왔습니다**. Amanda Askell이 Claude에 영혼을 후훈련시켰습니다 하하.

---

## Claude Code 제품이 기쁨을 주는 이유

모델 외에도, 분명히 Claude Code 제품이 마법 같은 경험을 만드는 데 큰 역할을 합니다.

제품으로서 Codex보다 QoL 기능에서 한참 앞서 있습니다. **하네스, 프롬프트, 그리고 모델이 마법 같은 경험을 만듭니다.** 모델은 놀랍지만, Claude가 하네스에서 편안하게 느끼고 함수 호출을 정확하게 만들기 위해 UX/UI와 코드/프롬프트에 들어간 **엄청난 양의 정교한 엔지니어링**이 있습니다. 이것은 이후 섹션에서 더 탐구할 것입니다.

---

## 이 글은 스폰서가 아닙니다

계속 가기 전에 - 제 이전 글이 어떻게든 Hackernews #5에 도달했고, 글이 Anthropic의 스폰서를 받았다는 혐의를 받았습니다. 진심이세요? Anthropic은 저 같은 무작위 사용자를 스폰서하지 않습니다. **Anthropic은 저에 대해 생각하지도 않습니다**(밈.jpeg) - 사용자 관점 외에는요.

칭찬 외에도, 저는 비꼬기도 했고, 장애에 대해 놀리기도 했고, Sonnet 4.5 슬롭에 대해 많이 놀렸습니다. 시간이 지남에 따라 느낀 것을 표현했고, 타임라인에서 좋은 담론으로 이어졌습니다.

이 모든 것을 말했지만, **Claude Code는 제가 경험한 가장 즐거운 제품 경험 중 하나였습니다.** 그 뒤에 있는 엔지니어링과 연구 팀에 감사하고 큰 존경을 표합니다.

수다는 이 정도로 충분합니다. 다음 몇 섹션에서는 이전 블로그에서 다루지 않은 유용한 기능들과 Claude 2.0 - 2.0.74 반복에서 도입된 주목할 만한 기능들에 대해 이야기하겠습니다.
