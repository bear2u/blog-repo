## Anthropic과의 사랑과 증오 관계, 그리고 Claude와의 화해 (힌트: Opus 4.5)

Opus 4.5 출시 이후 Claude Code 2.0과 함께 즐거운 시간을 보내고 있고, 그 이후로 제 메인 드라이버가 되었습니다. Claude에 대해 애정 어린 이야기를 하기 전에, 타임라인과 배경 이야기를 빠르게 살펴보겠습니다. 저는 블로그에서 수다 떠는 것을 좋아하고, 여기서 맥락을 설정하는 것이 중요하다고 생각합니다.

---

## 타임라인

2025년은 OpenAI와 Anthropic 모두 많은 프론티어 모델을 출시한 해입니다. 또한, 많이 언급되지 않지만 OpenAI가 실제로 코드 생성 능력에서 Anthropic을 따라잡았습니다 - 지능, 컨텍스트 윈도우 효과, 지시 따르기, 의도 감지 측면에서요.

| 시기 | OpenAI 출시 | Anthropic 출시 |
|------|------------|----------------|
| 2025년 상반기 | GPT-5, GPT-5-codex | Sonnet 4, Opus 4 |
| 2025년 하반기 | GPT-5.1, GPT-5.2 | Sonnet 4.5, Opus 4.5 |

Opus 4.5 출시 이후 45일도 채 되지 않은 시점에서 이 글을 쓰고 있습니다.

정말 격동의 해였고, 솔직히 말해서 2주마다 새로운 릴리스를 시도하느라 지쳤습니다.

GLM-4.7, Kimi-K2, Minimax-2.1 같은 여러 오픈 소스 경쟁자들도 있었습니다. 시장은 매우 경쟁적이고, 저-중 난이도 작업에 대해 저렴하지만 고성능인 중국 모델을 사용하는 사용자층이 분명히 있습니다.

그렇긴 해도, 저는 여전히 Anthropic/OpenAI가 중국 프론티어 모델보다 앞서 있다고 생각합니다. 후자는 DeepSeek R1 논문과 올해 초 Kimi K2 논문처럼 기술 오픈소싱 측면에서 더 많이 기여했습니다.

(참고: 개인적인 코딩 사용에 대해 이야기하는 것이지, 애플리케이션용 프로덕션 API 사용에 대한 것이 아닙니다.)

---

## 배경 이야기

### Claude와 우정 끝, 이제 Codex가 제일 친한 친구

6월 말부터 9월 초까지 Claude Code를 메인 드라이버로 사용했습니다. 9월 초에 Claude Max(월 100 USD) 구독을 취소하고 OpenAI Codex를 메인 드라이버로 전환했습니다. 전환은 두 가지 요인에 의해 주도되었습니다:

**첫째, Sonnet 4/Opus 4가 특별히 마음에 들지 않았고 GPT-5-codex가 Sonnet 4.5와 동등하게 작동하면서 훨씬 더 좋은 코드를 작성했습니다.**

**둘째, Anthropic에 API 장애가 많았고, 한때는 추론 버그로 인한 성능 저하도 있었습니다.** 이것도 여러 사람들이 차선책인 Codex나 Cursor의 GPT-5.1로 이동하는 주요 원인이었습니다.

또한 9월에는 시스템 설계와 사고 작업이 더 많았기 때문에 Claude Max 플랜(100 USD짜리)이 좋은 거래가 아니었습니다. Codex는 월 20 USD 구독만으로 엄청난 가치를 제공했고 거의 속도 제한에 걸리지 않았습니다. 추가로, Codex 개발자들은 버그를 푸시할 때마다 사용량 제한을 관대하게 리셋해줬습니다 ㅋㅋ.

---

## 나의 Codex 시대

10월 말까지 Codex(메인 드라이버)와 Cursor(취소 안 함)를 사용했습니다. Claude Sonnet 4.5가 9월 29일에 Claude Code 2.0과 함께 출시되었고... 다른 이메일 계정으로 20 USD 구독을 해서 시도해봤지만(프롬프팅 작업이 많았고 Claude 모델이 제 선호 선택이었기 때문에), GPT-5/GPT-5-codex가 느리긴 해도 전반적으로 더 나았습니다.

Sonnet 4.5의 문제는 빠르고 좋았지만, 버그로 이어지는 무분별한 변경을 많이 했다는 것입니다. 다시 말해, 나중에 GPT-5.1/GPT-5.1-codex에 비해 많은 슬롭(slop)을 생산한다고 느꼈습니다.

---

## Anthropic 리뎀션 아크 + 천명 회복

10월 30일쯤, Anthropic에서 구독을 취소한 사용자들에게 200 USD Max 플랜을 제안하는 이메일을 보냈고, 당연히 받아들였습니다.

Claude Code 사용량은 여전히 적었지만, 11월 24일에 Opus 4.5를 출시했고, 5일 동안 Opus 4.5를 시험해볼 수 있었습니다. 직장 일에 열심히 사용했고, 여러 기능을 발견하면서 이 기술적인 블로그를 작성했습니다.

### 왜 Opus 4.5를 사랑하고 전환한 이유

GPT-5.1로 전환했을 때도 비슷한 트윗을 했는데, 그때는 이 트윗의 절반 정도 반응이었습니다. 이는 당시 더 많은 사람들이 Opus 4.5에 공감했다는 것을 나타냅니다(적어도 트위터에서는). 또한, 많은 사람들이 GPT-5.1의 능력을 깨닫지 못했습니다.

SWE-bench-verified(코드 생성), Tau Bench(에이전트 작업) 같은 코딩 벤치마크에서 최고 수준이라는 것 외에도, Opus 4.5는 더 빨랐고, 코딩에서 동등했으며, 매우 협력적이고 커뮤니케이션이 좋았습니다. 이러한 요인들이 제 전환을 이끌었습니다. **좋은 바이브가 있었습니다.** 비교 포인트는 포스트 후반에서 더 다룹니다.
