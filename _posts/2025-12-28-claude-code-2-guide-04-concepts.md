## 비기술자를 위한 핵심 개념

기술적으로 가벼운(technically-lite) 분들이 이 글을 읽을 것이라고 가정합니다. 블로그 후반부 이해를 돕기 위한 몇 가지 개념을 설명합니다.

---

## 컨텍스트와 컨텍스트 윈도우

**컨텍스트(Context)**는 LLM에 제공되는 입력을 의미합니다. 보통 텍스트지만 요즘 모델들은 이미지, 오디오, 비디오도 지원합니다.

더 구체적으로, 컨텍스트는 **입력 토큰**입니다. **컨텍스트 윈도우**는 대화 중 LLM이 한 번에 보고 처리할 수 있는 **최대 토큰 양**을 의미합니다. 모델의 **작업 메모리**와 같습니다.

| 모델 | 컨텍스트 윈도우 | 대략적인 단어 수 |
|------|----------------|-----------------|
| Opus 4.5 | 200K 토큰 | ~150,000 단어 |
| GPT-5.2 | 400K 토큰 | ~300,000 단어 |
| Gemini 3 Pro | 1M 토큰 | ~750,000 단어 |

---

## 도구 호출 (Tool Calling)

도구 호출에 대해 배우세요. LLM이 텍스트를 생성할 수 있다는 것은 알지만, LLM이 **작업을 수행**하게 하려면 어떻게 해야 할까요? 이메일 초안 작성, 인터넷에서 날씨 조회, 구글 검색 등을 말입니다. 여기서 **도구**가 등장합니다.

**도구는 엔지니어가 정의한 함수**로, 이러한 정확한 작업을 수행합니다. 우리가 도구를 정의하고 시스템 프롬프트에서 LLM에게 알려주면, LLM이 대화 중 어떤 도구를 호출할지 결정할 수 있습니다! 도구 호출, 즉 작업이 수행되면, **결과가 LLM에게 다시 전달**됩니다.

```
도구 호출 흐름:
사용자 → LLM → 도구 호출 결정 → 함수 실행 → 결과 → LLM → 응답
```

## 에이전트 (Agent)

가장 간단한 정의는 **목표 달성을 위해 능동적으로 도구를 실행할 수 있는 LLM**입니다.

더 정교한 정의로는, Anthropic의 정의를 좋아합니다:

> "에이전트는 LLM이 자신의 프로세스와 도구 사용을 동적으로 지시하면서, 작업 완료 방식에 대한 제어를 유지하는 시스템입니다." - Building Effective Agents

```
┌─────────────────────────────────────────┐
│              에이전트 루프               │
│  ┌───────┐    ┌───────┐    ┌───────┐   │
│  │ 입력  │ →  │  LLM  │ →  │ 도구  │   │
│  └───────┘    └───┬───┘    └───┬───┘   │
│                   │            │        │
│                   └────────────┘        │
│                    (반복)               │
└─────────────────────────────────────────┘
```

---

## "에이전틱" (Agentic)

**모델의 도구 호출 능력**을 지칭합니다:

- 얼마나 **능동적**인지
- 도구 호출이 얼마나 **정확**한지
  - 작업을 수행하려는 사용자의 의도 감지
  - 올바른 도구 선택
  - 언제 멈춰야 하는지 아는 것

---

## 하네스/스캐폴딩 (Harness/Scaffolding)

Sonnet 4.5/Opus 4.5는 **모델**입니다. 이들이 반자율적인 방식으로 작동하려면 많은 **"스캐폴딩"**이 필요합니다:

- 코드 레이어
- 프롬프트
- 도구 호출
- 소프트웨어 패키징/환경

**참고:** Claude Code는 하네스가 아니라 **제품**입니다(TUI, 통합 등을 생각하세요). Claude Code는 하네스를 **가지고 있습니다**.

```
┌─────────────────────────────────────┐
│          Claude Code (제품)         │
│  ┌─────────────────────────────┐   │
│  │       하네스/스캐폴딩        │   │
│  │  ┌───────────────────────┐  │   │
│  │  │    Opus 4.5 (모델)    │  │   │
│  │  └───────────────────────┘  │   │
│  │  + 프롬프트                 │   │
│  │  + 도구 정의                │   │
│  │  + 코드 레이어              │   │
│  └─────────────────────────────┘   │
│  + TUI                             │
│  + 통합                            │
│  + 플러그인                        │
└─────────────────────────────────────┘
```
