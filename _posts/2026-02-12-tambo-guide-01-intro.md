## Tambo는 무엇인가

Tambo는 **React 앱에서 에이전트가 UI 컴포넌트를 선택해 렌더링**하도록 만드는 오픈소스 생성형 UI 툴킷입니다.
개발자는 컴포넌트와 프롭스 스키마를 등록하고, 사용자는 자연어로 요청합니다. 그 사이에서 Tambo가 스트리밍, 상태 관리, 도구 호출, MCP 연결까지 처리합니다.

README의 요약을 한국어로 옮기면:
- \"UI로 말하는\" 에이전트를 만들기
- React를 위한 오픈소스 생성형 UI 툴킷

---

## 어떤 문제를 푸나

일반적인 챗 UI는 "텍스트"로만 결과를 보여줍니다. 하지만 제품에서는 사용자가 원하는 결과가 **차트, 테이블, 대시보드, 폼, 작업보드** 같은 UI로 표현되는 경우가 많습니다.
Tambo는 이런 UI를 LLM이 상황에 따라 선택하도록 만들고, 프롭스를 생성해 실제 컴포넌트를 렌더링합니다.

README의 예를 한국어로 옮기면:
- \"지역별 매출을 보여줘\" -> `<Chart>` 렌더
- \"할 일을 추가해줘\" -> `<TaskBoard>` 업데이트

---

## 무엇이 포함되나

Tambo는 단일 라이브러리라기보다 "풀스택" 구성입니다.

- **React SDK**: `@tambo-ai/react` (프로바이더, 훅, 컴포넌트 등록, 도구/MCP 등)
- **백엔드**: 대화 상태(conversation state)와 에이전트 실행 루프를 담당
  - Tambo Cloud(호스팅)
  - 자체 호스팅(Docker)

---

## 핵심 기능 미리보기

- **컴포넌트 선택 자동화**: AI가 어떤 컴포넌트를 렌더할지 결정
- **프롭스 스트리밍**: 컴포넌트 프롭스가 점진적으로 채워지며 UI가 업데이트
- **상태형 UI(상호작용 컴포넌트)**: 한 번 렌더 후에도 사용자의 추가 요청으로 업데이트
- **로컬 도구 실행**: 브라우저에서 실행되어야 하는 함수(DOM, 인증 fetch 등)를 도구로 등록
- **MCP 통합**: Linear/Slack/DB 등 MCP 서버에 연결

## 이 시리즈에서 다룰 것

5. 모노레포 구조와 자체 호스팅 운영
