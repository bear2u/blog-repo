# 챕터 구조 가이드

## 권장 챕터 수
- 최소: 8개
- 권장: 10개
- 최대: 15개

## 표준 챕터 구성

### Part 1: 개요 (2-3개)
1. **소개 및 개요**
   - 프로젝트 소개
   - 핵심 특징
   - 기술 스택
   - 시리즈 목차

2. **전체 아키텍처**
   - 시스템 구조도 (ASCII 다이어그램)
   - 모듈 간 관계
   - 데이터 흐름

### Part 2: 핵심 모듈 (3-5개)
- 각 주요 모듈별 상세 분석
- 코드 구조 설명
- 핵심 클래스/함수 분석
- 예제 코드 포함

### Part 3: 인프라/유틸리티 (2-3개)
- 설정 시스템
- 유틸리티 함수
- 공통 컴포넌트

### Part 4: 활용 (1-2개)
- 실제 사용 예제
- 확장 방법
- 결론 및 참고 자료

## 챕터 제목 형식

```
{프로젝트명} 완벽 가이드 ({번호}) - {챕터 제목}
```

예시:
- UI-TARS 완벽 가이드 (1) - 소개 및 개요
- Claude Code 2.0 가이드 (4) - 핵심 개념

## 챕터별 권장 분량
- 최소: 200줄
- 권장: 300-500줄
- 코드 블록: 5-15개
